# Contributing to WA Housing Homelessness

Thank you for considering contributing to our project! We welcome contributions from everyone. Please follow the guidelines below to help us maintain a high standard of quality.

## Contribution Guidelines

1. **Fork the Repository**: Start by forking the project repository to your own GitHub account.
2. **Create a Branch**: From the main branch, create a new branch for your feature or fix. We recommend using descriptive branch names.

   ```bash
   git checkout -b feature/my-feature
   ```

3. **Write Tests**: If you are adding a new feature, please make sure to include tests that cover your new code.

4. **Lint and Format**: Ensure your code adheres to our code style standards.

5. **Submit a Pull Request**: Once your changes are ready, push your branch to your forked repository and submit a pull request to the original repository.

## Code Style Standards

We follow the following coding standards:

- **Language**: JavaScript
- **Indentation**: Use 2 spaces for indentation.
- **Quotes**: Use single quotes for strings.
- **Semicolons**: Always end statements with semicolons.

### Linting

Make sure to run the linter before submitting your changes:

```bash
npm run lint
```

## Instructions for Submitting Changes

1. Ensure all tests are passing.
2. Review your changes and ensure your commit messages are clear and descriptive.
3. Submit your pull request, including a description of the changes you made and why you made them.

Thank you for contributing to WA Housing Homelessness!
